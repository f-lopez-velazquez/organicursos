# Contribuir a OrganiCursos

Gracias por querer mejorar OrganiCursos.

## Antes de abrir un cambio

- revisa si ya existe un issue relacionado
- describe el problema real y el comportamiento esperado
- mantén cada pull request enfocado en una sola mejora o corrección

## Flujo recomendado

1. crea una rama desde `main`
2. instala dependencias con `npm install`
3. valida con `npm test` y `npm run build`
4. explica en el pull request qué cambió, por qué cambió y cómo se validó

## Estándares del proyecto

- prioriza claridad sobre complejidad innecesaria
- evita mezclar refactors grandes con arreglos puntuales
- documenta decisiones cuando afecten empaquetado, sidecars o compatibilidad multiplataforma
- conserva consistencia con la experiencia local-first del producto

## Reportes útiles

Cuando reportes un bug, intenta incluir:

- sistema operativo y versión
- versión de OrganiCursos
- pasos para reproducir
- resultado esperado
- resultado observado
- capturas o logs, si ayudan a aislar el fallo
